# MarkdownReader 多窗口需求文档

> 状态：待实现
>
> 目标版本：待定
>
> 创建日期：2026-07-14
>
> 产品模型：Word 式文档多窗口

## 1. 背景

MarkdownReader 当前使用 `WindowGroup` 承载主界面，但产品行为和内部控制链路仍按单窗口设计：应用会主动关闭多余主窗口，菜单命令主要通过全局 `NotificationCenter` 广播，文件打开、窗口恢复、拖拽、更新弹窗和 per-file undo 等流程也包含单窗口假设。

随着用户同时阅读多个 Markdown 文件或多个目录的需求增加，单窗口内反复切换会打断阅读位置和上下文。需要将应用升级为符合 macOS 与 Microsoft Word 使用习惯的文档式多窗口应用，同时保留现有三栏布局、阅读体验、Raw 编辑能力和文件安全语义。

本需求不是简单移除单窗口守卫，而是建立明确的窗口会话和命令路由规则，确保每个窗口可以独立工作，且不会发生跨窗口误操作。

## 2. 产品目标

1. 支持同时打开多个独立的 Markdown 文件或目录窗口。
2. 一个窗口维护一套独立的浏览、文档、编辑和视图状态。
3. 所有文档级命令只作用于当前活动窗口。
4. 同一 Markdown 文件在任意时刻只能由一个窗口会话持有，避免编辑、缓存、保存和 undo 冲突。
5. Finder、打开最近、拖拽、Dock 重开等入口在冷启动、热启动、有窗口和无可见窗口场景下行为一致。
6. 保持当前单窗口使用方式兼容；只打开一个窗口时，用户不应感知额外复杂度。

## 3. 非目标

首期明确不包含：

- 不支持 Excel 式“同一个 Markdown 文件同时显示在多个窗口”。
- 不支持多个窗口共同编辑同一文件。
- 不恢复上次退出时的完整窗口集合。
- 不实现窗口标签页管理、窗口平铺或自定义窗口管理器。
- 不实现跨窗口拖动文件、目录树节点或编辑内容。
- 不改变 Markdown 渲染、Raw 编辑、文件保存、主题和 Git 功能本身。
- 不重构为 `DocumentGroup`；是否采用该技术方案由实现设计决定，产品需求不作限定。
- 不新增云同步、协同编辑或跨设备会话同步。

## 4. 术语

### 4.1 窗口会话（Window Session）

一个主窗口及其独立运行状态。每个窗口会话具有稳定的唯一标识，并独立持有：

- 当前根目录或单文件模式；
- 当前选中文件；
- `AppViewModel`、`FileTreeViewModel`、`DocumentViewModel`；
- 文档内容缓存、脏状态和文件监控；
- 显示模式、滚动位置、查找状态和缩放状态；
- Sidebar、Outline、设置页、命令面板和全屏状态；
- 该会话使用的 undo/redo 历史。

### 4.2 资源

能够作为窗口打开目标的本地 Markdown 文件、纯文本文件或目录。

### 4.3 活动窗口

当前 key window。菜单快捷键和文档级命令必须路由到该窗口；隐藏窗口、非 key window 和系统 Panel 不得响应。

### 4.4 空白窗口

尚未打开文件或目录、未创建待保存 Untitled 文档的主窗口。空白窗口显示 Welcome 页面，可以被下一次打开操作复用。

## 5. 状态边界

### 5.1 应用级共享状态

以下状态在所有窗口之间共享：

- 语言、主题、字体、内容边距等用户偏好；
- 最近打开记录；
- 默认打开程序和命令行工具设置；
- 自动更新状态及更新检查结果；
- About 窗口；
- 窗口会话注册表和文件所有权注册表。

应用级设置发生变化时，所有窗口应按现有规则同步更新。

### 5.2 窗口级独立状态

以下状态必须按窗口隔离：

- 当前文件、根目录和目录树选中项；
- 内容、脏状态、Untitled 状态和保存面板状态；
- 文件监控和外部修改提示；
- 显示模式、缩放、滚动位置和光标位置；
- 查找替换状态及搜索结果；
- Sidebar、Outline、设置界面和命令面板显隐；
- PDF 导出目标和进行中状态；
- 全屏、拖拽悬停和错误提示状态；
- undo/redo 历史。

## 6. 核心交互规则

### 6.1 应用启动

| 场景 | 预期行为 |
|------|----------|
| 普通冷启动，未启用“恢复上次位置” | 创建一个空白窗口 |
| 普通冷启动，启用“恢复上次位置” | 创建一个窗口并恢复最后一次活动窗口的位置 |
| Finder 冷启动打开一个文件 | 创建一个窗口并显示目标文件，不额外创建空白窗口 |
| Finder 冷启动同时打开多个文件 | 每个不同文件创建一个窗口，不额外创建空白窗口 |
| 命令行一次传入多个文件 | 与 Finder 多文件打开规则一致 |

首期只恢复最后一次活动窗口，不恢复上次退出时的全部窗口。

### 6.2 新建窗口与新建文件

“新建窗口”和“新建文件”必须是两个不同概念：

- `File > New Window`：创建一个新的空白窗口。
- `File > New File`：在当前活动窗口创建 Untitled 文档，保持现有新建文件语义。
- 快捷键建议：
  - `Cmd+N`：保留现有“新建文件”行为；
  - `Cmd+Shift+N`：新建空白窗口。

当应用没有可见主窗口时执行“新建文件”，应先创建或恢复一个主窗口，再在其中创建 Untitled 文档。

### 6.3 打开文件或目录

通过 `Cmd+O`、菜单或打开按钮选择资源后：

1. 如果活动窗口是空白窗口，复用该窗口打开资源。
2. 如果活动窗口已经承载文件、目录或 Untitled 文档，为目标资源创建新窗口。
3. 不得静默替换其他已打开窗口的内容。
4. 打开操作不得影响非目标窗口的目录、选中项、滚动位置或编辑状态。

### 6.4 Finder、Open Recent 和外部打开

所有外部打开入口必须经过统一的窗口路由：

- Finder 双击；
- Finder 多选后打开；
- `File > Open Recent`；
- `mdr <path...>`；
- 应用收到的系统 open URL/file 事件；
- Dock 或其他应用请求 MarkdownReader 打开文件。

路由规则：

1. 目标资源尚未打开：优先复用一个空白窗口，否则创建新窗口。
2. 同一单文件资源已经由某窗口持有：激活并前置该窗口，不创建重复窗口。
3. 同一根目录已经作为目录窗口打开：激活并前置该目录窗口。
4. 一次收到多个不同资源：逐个路由，每个资源最多对应一个窗口。
5. 目标窗口被最小化或隐藏：恢复并前置正确的目标窗口，而不是任意“第一个隐藏窗口”。
6. 路径不存在或类型不受支持：只在请求发起或被选定的目标窗口显示错误，不影响其他窗口。

### 6.5 目录窗口内导航

- 在目录树中点击 Markdown 文件，继续在当前目录窗口中切换内容，不自动创建新窗口。
- 窗口切换文件后，应保留该窗口现有的 per-file 内容缓存、显示模式和 undo 语义。
- 两个不同目录窗口可以同时存在，包括路径有包含关系的目录。
- 即使两个目录范围重叠，同一具体文件也不能同时由两个窗口取得可编辑所有权。
- 如果用户点击的文件已由其他窗口持有，目录窗口不得先修改有效选中项或加载文件；应保持当前文档和当前有效选中项不变，并激活文件所有者窗口。
- 已由其他窗口持有的文件应在目录树行尾显示“已在另一窗口打开”标识，并提供本地化 tooltip/accessibility label。

### 6.6 同一文件所有权

为避免 Raw 编辑冲突，应用必须维护文件所有权：

1. 一个规范化文件 URL 在任意时刻最多归属一个窗口会话。
2. 所有权范围包含窗口当前文件，以及该窗口仍保留未保存缓存或 undo 历史的文件。
3. 其他窗口请求打开已被持有的文件时，应激活所有者窗口，并在必要时让该窗口选中目标文件。
4. 文件保存、另存为、重命名或移动后，所有权必须原子地迁移到新 URL。
5. 文件被关闭、放弃或窗口会话销毁后，必须释放相应所有权。
6. 路径比较应使用规范化 URL，避免相对路径、符号链接或大小写差异绕过去重。

首期不提供“仍然打开副本”或“只读副本”选项。

### 6.7 链接打开

- 点击当前根目录内的 Markdown 链接：沿用现有行为，在当前窗口导航。
- 点击根目录外的 Markdown 链接：首期仍在当前窗口打开，保持现有交互，不自动创建新窗口。
- 如果目标文件已归属其他窗口：激活所有者窗口，不在当前窗口重复打开。
- HTTP/HTTPS 等外部链接继续交给系统默认浏览器。

### 6.8 拖拽打开

- 拖拽文件或目录到某个窗口，只能影响鼠标所在的目标窗口。
- 目标窗口为空白时，直接在该窗口打开。
- 目标窗口已有资源时，创建新窗口打开拖入资源，避免覆盖当前上下文。
- 拖入已由其他窗口持有的文件时，激活所有者窗口。
- 拖拽悬停效果和不支持类型提示不得在其他窗口同步出现。

## 7. 菜单与命令路由

### 7.1 窗口级命令

以下命令只能由活动窗口处理：

- 新建文件、保存、另存为、重新加载；
- 打开文件、打开目录、打开最近；
- 导出 PDF；
- Sidebar 和 Outline 显隐；
- Rendered/Raw 模式切换；
- 放大、缩小、恢复缩放；
- 查找、查找下一个、查找上一个、查找替换；
- 命令面板和窗口内设置界面；
- 关闭、最小化、缩放和全屏；
- 文件树操作、Markdown 链接打开和拖拽反馈。

非活动窗口即使仍在视图层监听应用事件，也不得执行上述命令或显示面板。

### 7.2 应用级命令

以下命令只执行一次，不随窗口数量重复：

- About；
- 自动检查和手动检查更新；
- 清除最近打开记录；
- 应用退出；
- 全局用户偏好变更。

更新检查不得因创建多个窗口而重复发起；更新弹窗应绑定到当前活动窗口，或由单一应用级窗口展示。

### 7.3 Window 菜单

使用或补全 macOS 标准 `Window` 菜单，至少支持：

- 显示当前所有主窗口；
- 选择条目后激活对应窗口；
- Minimize；
- Zoom；
- Bring All to Front。

窗口菜单中的标题应能区分不同文件、目录和 Untitled 窗口。

## 8. 窗口生命周期

### 8.1 关闭单个窗口

- `Cmd+W` 和自定义关闭按钮只关闭活动窗口。
- 关闭一个窗口不得清理或改变其他窗口的文档、文件监控、undo 或界面状态。
- Untitled 且存在未保存修改时，继续显示“保存 / 不保存 / 取消”。
- 选择取消后，仅阻止当前窗口关闭。
- 选择保存后，应等待当前窗口保存成功再关闭。
- 选择不保存后，应完整清理当前窗口的 Untitled 状态并释放文件所有权。

### 8.2 关闭最后一个窗口

- 关闭最后一个主窗口不等于退出应用，沿用标准 macOS 行为。
- 应用保持运行时，Dock 点击应创建或恢复一个空白窗口。
- Finder 或 Open Recent 请求应创建/恢复正确窗口并打开目标资源。
- 不得出现“文件已经加载但窗口不可见”的状态。

### 8.3 退出应用

- 退出时应检查所有窗口中的未保存 Untitled 文档。
- 每个有风险的窗口必须获得明确处理结果；不得只检查 key window。
- 任一窗口选择取消，应取消本次退出。
- 退出过程不得重复弹出同一窗口的保存面板。

### 8.4 隐藏、最小化与全屏

- 恢复目标资源时必须定位其所属窗口，不得默认激活窗口数组中的第一个窗口。
- 一个窗口进入或退出全屏，不得改变其他窗口的全屏布局状态。
- 自定义红绿灯按钮必须操作按钮所在窗口，不能依赖可能已经变化的全局 `NSApp.keyWindow`。

## 9. 编辑、保存与 Undo

### 9.1 编辑状态隔离

- 每个窗口独立维护当前内容、磁盘快照、脏状态、外部修改状态和文件监控器。
- 任一窗口编辑或重新加载文件，不得修改其他窗口的 `DocumentViewModel`。
- 外部修改提示只显示在持有该文件的窗口。

### 9.2 Undo/Redo 隔离

- UndoManager 必须按窗口会话和文件隔离。
- `Cmd+Z`、`Cmd+Shift+Z` 和 Edit 菜单验证必须使用当前活动窗口当前文件对应的 UndoManager。
- 窗口切换、文件切换或某个编辑器销毁，不得清空其他窗口的 undo 历史。
- 不允许使用单个应用级“当前活跃 UndoManager”作为所有窗口的唯一来源。

### 9.3 保存与另存为

- 保存命令只保存活动窗口的当前文件。
- 保存面板必须附属于发起操作的窗口，且同一窗口内保持重入保护。
- 另存为成功后，同步更新窗口标题、最近记录和文件所有权。
- 如果另存为目标已被其他窗口持有，应阻止覆盖并提示用户激活对应窗口或选择其他路径。

## 10. 窗口状态与恢复

首期使用受控的单窗口恢复策略：

- 继续保存最后一次活动窗口的文件或目录位置。
- 应用正常重启时最多恢复一个窗口。
- 不保存和恢复其他窗口的布局、尺寸、屏幕位置或文档集合。
- 多窗口会话恢复作为后续独立需求，不与首期绑定。

在实现多窗口后，原有 `lastOpenedFile` 和 `lastOpenedDirectory` 的含义应明确为“最后一次活动窗口的位置”，不能由后台窗口的状态变化任意覆盖。

## 11. 性能与稳定性要求

- 创建新窗口不得重复启动应用级后台任务，例如自动更新检查。
- WebView 预热属于应用级行为，只执行一次；每个文档窗口可以拥有自己的实际渲染页面。
- 多个空闲窗口不得持续重复扫描目录或重复处理同一全局通知。
- 同时打开 5 个包含普通 Markdown 文件的主窗口时，应用应保持可交互，无崩溃、死锁、保存面板风暴或明显持续 CPU 占用。
- 关闭窗口后，其文件监控、定时器、Notification observer、WebView 回调和 undo 引用必须释放。
- Swift 6 严格并发下，不得通过新增 `nonisolated(unsafe)` 全局可变状态规避窗口隔离。

## 12. 错误处理

- 窗口创建失败时，保留原窗口内容并显示一次可理解的错误，不得丢失未保存内容。
- 目标文件不存在、权限不足或类型不支持时，只影响目标窗口。
- 文件所有权迁移失败时，不得出现两个窗口同时认为自己持有同一文件。
- 多个外部打开事件连续到达时，路由必须幂等，不得生成重复窗口。
- 隐藏窗口、系统 Panel、About 窗口和 PDF 导出使用的离屏窗口不得被识别为文档主窗口。

所有新增面向用户的提示必须通过 `L10n` 提供简体中文、繁体中文和英文文本。

## 14. 实现约束与重点影响区域

实现必须优先处理以下现有单窗口假设，但本需求不限定具体类名或最终代码结构：

1. `AppDelegate` 中关闭多余窗口、激活第一个隐藏窗口和阻止系统重开窗口的逻辑。
2. `MarkdownReaderApp` 中通过全局通知发送菜单命令和打开最近事件的逻辑。
3. `ContentView`、`DetailView`、`WebViewMarkdownView` 中所有未按目标窗口过滤的通知监听。
4. 全局 `_activePerFileUndoManager`、`UndoManagerProvider.shared` 及 `NSWindow.undoManager` swizzling 的窗口隔离。
5. `OpenPanelHelper.isPanelShowing` 等应用级面板重入状态是否应改为按窗口管理。
6. `SettingsModel.shared` 中全局偏好与最后活动位置字段的职责边界。
7. 自动更新、WebView 预热、文件拖拽 overlay 和窗口关闭代理的生命周期。
8. 外部文件事件当前只处理第一个 URL 的限制。

不得通过在每个通知处理器前零散增加 `isKeyWindow` 判断来完成全部改造。需要形成统一的窗口注册、资源路由和命令分发边界，避免后续新增命令再次广播到所有窗口。

## 15. 验收标准

### 15.1 基础多窗口

- [ ] `Cmd+Shift+N` 可以连续创建多个空白窗口。
- [ ] 不再存在会主动关闭合法第二个主窗口的单窗口守卫。
- [ ] 两个窗口可以分别打开不同文件或目录，并独立切换、滚动和编辑。
- [ ] 关闭任一窗口不会影响其他窗口。

### 15.2 命令隔离

- [ ] 在窗口 A 执行保存、查找、缩放、显示模式切换或导出，窗口 B 不响应。
- [ ] Sidebar、Outline、设置页、命令面板和全屏状态互不串扰。
- [ ] 自定义红绿灯始终操作按钮所在窗口。
- [ ] 手动检查更新只出现一个结果界面；自动检查不会随窗口数增加而重复请求。

### 15.3 文件打开路由

- [ ] Finder 冷启动打开一个文件时只出现一个正确窗口。
- [ ] 应用运行时从 Finder 打开新文件时创建或复用正确窗口，不覆盖已有文档窗口。
- [ ] Finder 一次打开多个文件时，每个不同文件只出现一个窗口。
- [ ] Open Recent、`Cmd+O`、拖拽和 CLI 入口遵循同一去重规则。
- [ ] 再次打开已经持有的文件时激活原窗口，不创建副本。
- [ ] 单文件窗口已打开 `A.md` 时，目录窗口点击 `A.md` 会保持目录窗口原文档和有效选中项不变，并激活单文件窗口。
- [ ] 目录树能够标识由其他窗口持有的文件；所有者窗口关闭后标识消失，文件可在目录窗口正常打开。
- [ ] 所有窗口关闭后，通过 Finder、Open Recent 或 Dock 仍可恢复可见窗口。
- [ ] 不出现隐藏窗口收到事件、文件已加载但界面不可见的问题。

### 15.4 编辑安全

- [ ] 两个窗口的 undo/redo 历史完全独立。
- [ ] 在窗口 A 切换文件或关闭编辑器，不会清空窗口 B 的 undo 历史。
- [ ] 同一文件不能同时进入两个窗口的可编辑状态。
- [ ] 另存为、重命名和移动后，文件所有权正确迁移。
- [ ] Untitled 保存/不保存/取消仅影响所属窗口。
- [ ] 退出应用时会覆盖所有窗口中的未保存 Untitled 状态，取消可阻止退出。

### 15.5 稳定性与回归

- [ ] `swift build` 通过。
- [ ] 新增窗口路由和文件所有权相关测试目标及自动化测试。
- [ ] 连续创建、切换和关闭 20 次窗口无崩溃、僵尸窗口或重复 observer 行为。
- [ ] 同时保留 5 个普通文档窗口时无保存面板风暴、明显持续 CPU 异常或跨窗口状态污染。
- [ ] 现有单窗口打开目录、打开单文件、Finder 双击、Open Recent、拖拽、保存、导出 PDF 和关闭后重开流程全部回归通过。

## 16. 建议测试矩阵

| 维度 | 场景 |
|------|------|
| 启动状态 | 冷启动、热启动、应用运行但无可见窗口 |
| 打开入口 | `Cmd+O`、Open Recent、Finder、CLI、拖拽、Markdown 链接 |
| 资源类型 | 单文件、目录、Untitled、多个文件、重复文件、重叠目录 |
| 窗口状态 | key、非 key、隐藏、最小化、全屏、已关闭 |
| 编辑状态 | 未修改、普通脏文件、Untitled 脏文件、外部修改、保存中 |
| 命令类型 | 保存、另存为、查找、缩放、显示模式、导出、设置、更新 |
| 生命周期 | 新建、切换、关闭一个、关闭最后一个、Dock 重开、退出应用 |

至少应为以下纯逻辑能力建立自动化测试：

- 窗口会话注册与注销；
- 文件 URL 规范化和所有权去重；
- 多 URL 打开事件的路由决策；
- 空白窗口复用规则；
- 活动窗口命令分发；
- 另存为、重命名、移动后的所有权迁移；
- 窗口关闭后的状态释放。

## 17. 文档同步要求

功能实现完成后，需要同步更新：

- `docs/requirements.md`：移除“单窗口应用”和“多窗口（首版）”排除项，新增 MW 系列需求状态；
- `docs/architecture.md`：补充窗口会话、命令路由和应用级/窗口级状态边界；
- `docs/design.md`：补充 New Window、Window 菜单和窗口行为；
- `CHANGELOG.md`：记录多窗口支持及外部打开行为变化；
- 对应版本的 release notes。

## 18. 完成定义

只有在以下条件全部满足后，才能认为多窗口功能完成：

1. P0 功能需求全部实现并通过验收。
2. 不存在合法第二窗口被自动关闭或全局通知驱动多个窗口误操作的问题。
3. 同一文件单窗口所有权和 undo 隔离经过自动化测试验证。
4. 冷启动、热启动、无可见窗口和多文件外部打开均通过人工回归。
5. `swift build` 及新增测试全部通过。
6. 相关架构、需求、设计、变更记录和发布说明已同步。
